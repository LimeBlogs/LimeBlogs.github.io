一次KernelSU/SukiSU折腾记录

2026-07-13T06:45:27.000Z2026-07-14T05:31:31.411Z2.5k中国大陆

为什么要用KernelSU/SukiSU?

笔者在自己的远古设备Redmi12C上一直使用Magisk的root方案，确实稳定，最近一直在折腾海外虚拟卡/实体卡。单拎出Wise而言，刚装上就提示不支持在已Root的设备上运行，其他的银行直接卡在注册页，无法进入，Ifast更是直接卡崩。
呵呵了，我觉得面具已经没办法完成我的日常需求了，目光自然而然转向更新的Root方案，终究是老骥伏枥，不胜当年。
说真的，面具过过国内的银行还算轻松，但将目光放到海外的银行就是一大坨了。基本上都进不去。 KernelSU方案的隐藏性我还是认可的，除了比较挑设备外基本无敌，其知名分支SukiSU更是牛大了，闲话就说这么多，接下来来好好说说我是如何刷入的。希望能帮到在搜索引擎大海捞针的各位。

提示

请先确保设备已解锁Bootloader，并且在开始前备份好原厂boot避免设备变砖。

关于内核

开始请先确保自己系统在互联网上存在内核源码，本次构建使用Lineage-21演示，Github fork到自己的账户下。

感谢

感谢Gemini/DeepSeek，期间的报错都是他修复的，为我节省了不少时间。

纳尼？我的设备不支持？

刷Ksu的第一步永远是看自己的设备是否支持GKI，不幸的时，我不支持。


这倒不是什么大问题，之前好歹也编译过OrangeFox，多少有点把握，依照Ksu的官方文档来看，只需要编译就好了。

如果应用程序显示 “不支持”，则表示您的设备不支持 KernelSU，你需要自己编译设备的内核才能使用，KernelSU 官方不会也永远不会为你提供一个可以刷写的 boot 镜像。

确定方案

构建这种东西，首先必要的就是Linux，如果我的主力机是Windows那么则为WSL。但考虑到要拉取的东西实在有点多，梯子也吃不消，我也不想把我的电脑搞得乱七八糟，因此直接走Github Action就成为了我的首选。

但Github的缺点也很明显，构建内核这种事情很难说一次成功，期间多少会遇到报错，你要查日志啥的还是挺麻烦的。

准备工作

  • 已解锁BL的手机
  • 内核源码，因为笔者系统是LineageOS，在github上可以轻松找到
  • 一个好心态

开始

我这个LineageOS比较老了，如上图所示，内核版本只有4.19，这个内核版本其实非常危险，太几把老了。
我也是后来在网上搜索才知道SukiSU貌似不支持这个内核版本，至少在我看来，即便README.md写了兼容4.14+(需自己编译)，但是以我折腾的结果来看，其困难程度是非常大的，单靠Github Action非常困难。因此不考虑SukiSU，整了也肯定会报错的别问我怎么知道的。

接下来就要把目光转向KernelSu，好在其生态已非常丰富，在搜索引擎随便一搜就能找到不少现成的工作流。这里简单列出来我试过的几个。

Github - /xiaoleGun/KernelSU_actionhttps://github.com/xiaoleGun/KernelSU_action


Github - /Numbersf/Action-Buildhttps://github.com/Numbersf/Action-Build


好吧，原本我试过好几个，但其实有价值的就这个两个，还有些Fork完试了一下就删了。我使用的设备树是LineageOS的开源成品，android_kernel_xiaomi_earth同理，你也应该去找一下自己的内核源码。
接着，将目光转到Action，我们需要来配置一下Action。

插播一条消息


在构建了十几次依旧报错无法解决后，我心灰意冷，本来不报任何希望。接着我就在搜索引擎搜索，万一，我是说万一，万一真的有前人构建过了呢？


果不其然，还真是。


我看到Release的时候人都傻了，我千辛万苦挖报错，问AI，一天几十次在Action发呆。结果已经有成品了？这是个悲伤的故事，我们在做任何事前都要提前在网上找好资料。。万一有成品就不用重复造轮子了。这个是给KernelSU-Next做的适配，我是Lime，这个内核非常正常稳定，还是支持susfs的。。。不过这个作者使用的内核源码与我相同，只有分支不一样，我只需要稍微参考一下就知道自己到底输在哪。
